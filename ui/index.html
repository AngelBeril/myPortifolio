API Security Research

Hybrid AI anomaly detection for Kubernetes API server logs (Isolation Forest + LSTM).

View details

This research focuses on securing Kubernetes API servers by detecting abnormal or malicious access patterns in API request logs. The Kubernetes API server is the central control point of the cluster, and any unauthorized or unusual activity (e.g., privilege escalation, token misuse, lateral movement, brute-force access) can compromise the entire environment. Traditional rule-based security systems often fail to detect evolving and unknown attacks. Therefore, a hybrid AI approach is used to improve detection accuracy. Concept The system combines: Isolation Forest (IF) — a machine learning model that identifies statistical outliers in numerical API log features (e.g., request frequency, response size, latency). Long Short-Term Memory (LSTM) — a deep learning model that learns sequence behavior patterns over time (e.g., order of API calls per user/role). By using both models, the system detects: Irregular request frequency spikes Unusual API endpoint access by users or service accounts Suspicious command sequences indicating potential attacks
